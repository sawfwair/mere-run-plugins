# Code of Conduct

## Our Standard

This project is for people building local-first `mere.run` companion tooling.
Public discussion should stay practical, respectful, and focused on helping
contributors and users understand the software.

Examples of welcome behavior:

- being direct without being dismissive
- reporting bugs with enough context to reproduce them
- respecting user privacy, provider credentials, and paid-resource safety
- keeping disagreements technical and specific

Examples of unwelcome behavior:

- harassment, threats, insults, or personal attacks
- publishing private credentials, account identifiers, or non-public user data
- pushing contributors to run paid resources without clear cost and cleanup notes
- derailing issues or pull requests with unrelated arguments

## Reporting

Report conduct concerns privately to security@sawfwair.com. Include links,
screenshots, or command output when that helps, but do not include secrets.

Maintainers may remove comments, close issues, block accounts, or restrict
participation when behavior makes the project unsafe or unproductive.
